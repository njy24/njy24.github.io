Formal Definition: What Exactly Is Shell?

Def: Shell is a Command-Line Interpreter(CLI). It's a interface program between users and the operating system kernel. It's actually an **ordinary application** that runs dependent on the kernel and cannot exist dependently of the operating system

Why it matters

  1. The Bridge Connecting Users and the Kernel
  2. The Core Tool for Automated Operation and Maintenance and Development
  3. An Indispensable Means for Server and Remote Management
  4. Improve Work Efficiency

How it works (principles / mechanism)

  1. Read input: Get your command.
  2. Parse & split: Separate command, argument, symbols.
  3. Locate program: Search via PATH environment variable.
  4. Create child process: Call system to run it.
  5. wait for excution: Kernel runs and return output.
  6. Display result: Shell formats and prints output.
  7. Return to prompt: Wait for next command.
