版本控制简述

what

需求：管理许多人开发的不同的代码，需要对代码进行版本控制

重要性：管理程序文档，提高协作效率。多人长时开发。

版本控制：

对软件制品的更改和管理，他是软件配置管理的重要组成部分

  1. 组织保护软件制品：源代码以及文档
  2. 实现跨区域、多人的协同开发
  3. 记载和追踪一个或多个文件的历史记录
  4. 跟踪记录软件开发过程
  5. 统计软件开发工作量

基线管理

  1. 一个人之成为一个代码线
  2. 一条代码先的上级称为他的基线
  3. 主线是没有基线的代码线
  4. 版本控制就是管理代码线、基线和主线的过程

产品版本号命名

  1. 每一个软件发布的版本都需要分配唯一的版本号以明确表示不同的发布版本
  2. 常用的点分式版本命名规范M.S.F.B.([SP][C])[]表示可选字段
    1. 主版本号M：表示产品平台或整体架构
    2. 次版本号S：表示局部架构、重大特性或无法向前兼容的接口
    3. 功能版本号F（特性）：表示规划的新功能版本
    4. 编译版本号B：标识编译构建的版本号
    5. 补丁包版本号SP：把一段时间的补丁打包出来一个补丁包
    6. 补丁版本号C：标识一个补丁

应用场景

  1. 代码演化历史追踪
  2. 历史版本回退
  3. 多人开发协作冲突
  4. 代码质量检查

分布式版本控制思想

  1. 集中式：版本库->开发者
    1. 高度集中，难以支持协同开发
    2. CVS，SVN，ClearCase
  2. 分布式：本地仓库&&中央仓库
    1. 不依赖中央服务器、可在本地开发、有效支持协同开发
    2. Git

Git是啥

  1. git式一种分布式的版本控制系统
  2. 由Linus Toralds研制开发用于帮助管理Linux内核开发的一个开源的版本控制系统软件
  3. Git是目前世界上最先进的分布式版本控制
  4. git采用分布式的管理模式
    1. 本地拥有完整的版本库，支持离线操作
    2. 应对中心服务器的崩溃
    3. 辅助代码审查、持续集成
  5. 基本功能:
    1. 版本管理
    2. 过程管理
    3. 代码评审
    4. 拓展功能

Git咋用

基本概念

  1. 仓库（Repository）：软件所有文件的完整修订历史
  2. 版本（revision）：代码库的编号方案，如Tag 0.1
  3. 分支（Branch）：对代码库并行修改时的代码库副本，如：master（主分支），develop（），branch1…
  4. 提交（commit）：对分支的修订
  5. 下拉（pull）：将远程的一个分支代码读取并保存到本地
  6. 推送（push）：将本地分支代码发送到某个分支
  7. 合并（merge）：将对相同文件在不同问分支的修改合并到同一个分支中
  8. 冲突（conflict）：当两个分支存在对同一文件的不同修改并试图合并两个分支，则会发生冲突。

git的使用

1. 分支管理流程：初始化过程（主要负责人）

  1. 创建版本库（默认master分支），push初始代码
  2. 创建远程develop分支

2. 分支管理流程：基本过程（成员）

  1. 克隆代码，切换到develop分支
  2. 新建自己的分支my_branch（组员之间分支名不同）
  3. 修改my_branch：正常开发，修改完成后将修改的内容推送到远程的my_branch分支

3. 分支管理流程：合并分支（组长）

  1. 切换develop分支，pull代码
  2. 合并组员分支到devlope分支
  3. 解决冲突，将合并好的develop分支推送到远程
  4. 将develop分支合并到master分支

git命令

  1. git init
    1. 执行过后，git仓库生成.git目录，包括了资源的所有元数据
    2. 使用当前目录作为git仓库，直接在当前目录进行初始化
    3. 使用指定目录作为git仓库，在初始化命令之后加上置顶目录路径，如git init newrepo
  2. git clone <repo> <directory>
    1. 将远程仓库中的代码拷贝项目到本地仓库
    2. 其中repo表示远程仓库目录，diretory代表本地仓库目录
  3. git add [file1] [file2]... or [dir]添加文件到暂存区
  4. git checkout -- [file1]或者git restore [file1]撤销不需要的修改（暂存区内的）
  5. 提交修改到本地仓库
    1. git commit -m [messsage]
    2. git commit [file1] [file2] ... -m [message]将暂存区内容添加至本地仓库
    3. git log 查看commit提交日志
  6. git remote add [远程仓库名] [远程仓库地址]添加远程版本库
  7. git push [远程仓库名] [本地分支名] [远程分支名]推送本地内容到远程仓库
    1. 推送本地内容时，会将所有未推送至远程仓库的内容，都提交到远程仓库
  8. git pull 远程主机名 远程分支名:本地分支名
    1. 别人对项目进行了修改，本人需要将修改合并到本地
  9. git pull 远程主机名 远程分支名：本地分支名 -f
    1. 如果远程分支和本地分支对同一内容进行了修改，将导致和并发生冲突
    2. 可以进行强制拉取，使用远程分支的修改，覆盖本地分支的修改，用-f参数
